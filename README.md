# LiveVoiceBridge

LiveVoiceBridge は、YouTube Live などの配信チャット（コメント）をリアルタイムに取得し、多様な音声合成（TTS）エンジンを介して読み上げるためのデスクトップアプリケーションです。
視聴者からのコメント内容に応じて読み上げ音声の速度・音量・キャラクターを動的に変更したり、効果音（SE）を鳴らしたりするインタラクティブな機能や、効率的な音声再生を実現する高度なキャッシュシステムを備えています。

---

## 主な特徴

- 🎙️ **多彩な音声合成（TTS）エンジンへの対応**
  - VOICEVOX、COEIROINK、棒読みちゃんに加え、ローカルで動作する高品質な Supertonic / Supertonic Lightweight に対応。
- 💬 **コメント内特殊コマンドによる動的制御**
  - コメント中に特定のテキストを含めることで、速度・音量・音程・話者の変更、効果音の再生、さらにはユーザー辞書の教育・忘却が可能です。
- ⚡ **高度な WAV キャッシュシステム**
  - SQLite3 データベースを用いた使用統計分析に基づき、よく読まれる言葉を自動でキャッシュ化（一時・永続キャッシュ）し、TTS の処理負荷と発話遅延を極限まで低減します。
- 🎛️ **音声エフェクト処理**
  - 音声波形を直接加工し、エコー、山びこ（3回ディレイ）、パンニング（左・右・両耳）効果を適用できます。
- 📺 **ピクチャーインピクチャー（PiP）コメント表示**
  - コメント一覧部分を半透明かつ最前面表示が可能な独立ウィンドウとして分離（ポップアウト）できます。
- 🛠️ **Windows 環境における日本語パス問題の自動回避**
  - Windows のユーザーフォルダ名に日本語（マルチバイト文字）が含まれている場合に発生する `pyopenjtalk` の初期化バグを、起動時に自動で一時ディレクトリ（TEMP）へアセットを退避することで自動回避します。

## 技術スタック

- **GUI フレームワーク**: PySide6 (Qt for Python)
- **音声処理**: Python 標準 `wave`, `struct` ライブラリ（エフェクト自製）、`winsound` (Windows) / `aplay`・`pw-play`・`paplay` (Linux) / `afplay` (macOS)
- **データベース**: SQLite3 (キャッシュメタデータ管理)
- **通信**: gRPC (YouTube 配信チャットのストリーミング取得)

## 対応音声合成（TTS）エンジン

設定画面または `config.json` にて、以下のいずれかのエンジンを選択できます。

1. **VOICEVOX / COEIROINK**
   - 外部で起動しているローカルサーバー（デフォルト: VOICEVOX `http://127.0.0.1:50021` / COEIROINK `http://127.0.0.1:50032`）に HTTP リクエストを送信して音声を生成します。
   - アプリ側から実行ファイル（`.exe`）のパスを指定して自動起動させることも可能です。
2. **棒読みちゃん**
   - Socket 通信（デフォルト: `127.0.0.1:50001`）を介して棒読みちゃん本体に音声生成と再生を指示します。
3. **Supertonic Lightweight**
   - `sherpa-onnx` バックエンドを使用したローカル音声合成エンジン。軽量で高速に動作し、CPU / GPU / NPU に自動で最適化されます。
   - *※モデルファイルは設定画面から直接ダウンロード可能です。モデルのダウンロードおよび音声アセットの利用にあたっては、ダウンロード時にライセンス（Apache-2.0等）の同意確認が行われます。詳細は [sherpa-onnx プロジェクト](https://github.com/k2-fsa/sherpa-onnx) をご確認ください。*
4. **Supertonic**
   - `supertonic-3` モデルと専用 SDK を使用した高品質なローカル音声合成エンジン。推論ステップ数（`num_steps`）の調整や、CPU/GPU、OpenVINO 経由での Intel NPU 推論に対応しています。

## コメント内特殊コマンド

視聴者が配信チャットで以下の書式を使ってコメントすると、読み上げ動作やシステム辞書を動的にコントロールできます。

### 1. ユーザー辞書登録（教育・忘却）
- **教育**: `教育(単語=よみがな)`
  - 例: `教育(LiveVoiceBridge=ライブボイスブリッジ)`
  - 効果: 辞書の「配信コメント」グループ（`dict/配信コメント.json`）に対象の単語が登録され、以降その単語は指定されたよみがなで読み上げられます。
- **忘却**: `忘却(単語)`
  - 例: `忘却(LiveVoiceBridge)`
  - 効果: 「配信コメント」グループから対象の単語を削除します。

### 2. 効果音再生（SE）
- **SE再生**: `再生(ファイル名)` または `音(ファイル名)` または `sound(ファイル名)`
  - 例: `再生(warning.wav)` / `sound(cheer.wav)`
  - 効果: アプリケーション実行ディレクトリにある `sounds/` フォルダ内から対象のオーディオファイルを検索し、コメント読み上げと同時に再生します。
  - *※セキュリティのため、パス指定は無視されファイル名部分のみが抽出されます。*

### 3. 音声パラメータ調整
コメントの途中で以下のコマンドを挿入すると、それ以降のテキストの読み上げ属性が変更されます。（1つのコメント内で複数回変更可能です）

| コメント記述 | 対象パラメータ | 設定値と変換仕様 |
| :--- | :--- | :--- |
| `速度(数値)` | 話速（スピード） | 指定値 / 100 倍（例: `速度(120)` は 1.2 倍速） |
| `音程(数値)` | 音高（ピッチ） | `(指定値 - 100) / 100 * 0.15` の式でピッチ差分を算出 |
| `音量(数値)` | 音量（ボリューム） | 指定値 / 100 倍（例: `音量(80)` は 80% の音量） |
| `声(数値)` | 話者（キャラクター） | TTS エンジン側の Speaker ID を指定値に変更 |
| `エコー(数値)` | エコー効果 | エコーの減衰率を設定（数値が大きいほど響きが強くなります） |
| `やまびこ(数値)` | 山びこ効果 | 山びこ（3回の減衰ディレイ）の減衰率を設定 |

### 4. パンニング（定位）
コメントの文頭や途中でパンニングコマンドを挟むことで、左右の定位を振ることができます。

- `左)` または `左）`: 音声を左チャンネルのみから再生します。
- `右)` または `右）`: 音声を右チャンネルのみから再生します。
- `両)` または `両）`: 音声を両チャンネル（ステレオ）に戻します。

## 高度な WAV キャッシュシステム

本アプリは、読み上げ音声のリアルタイム生成負荷を抑えるために、SQLite3 データベースを活用した高度な音声キャッシュ機構（`TtsWavCache`）を備えています。

### キャッシュの昇格・プロモートロジック
入力テキストは自動で分類され、出現頻度や利用期間などの統計情報に基づいてキャッシュレベル（無 / 一時 / 永続）が自動的に管理されます。

- **定型句 (`fixed_phrase`)**:
  - 対象: `コメントありがとうございます。` や `初見さん、いらっしゃい。` などの定型文。
  - ロジック: 初回再生時から即座に **永続キャッシュ (`persistent`)** として保存されます。
- **名前 (`name`)**:
  - 対象: 文末が `さん` や `ちゃん` などの敬称で終わるテキスト。
  - ロジック: 2日以上または2セッション以上検出された場合に **永続キャッシュ** に昇格します。
- **短いリアクション (`short_reaction`)**:
  - 対象: 2〜20文字の短いテキスト（例: 「なるほど」「草」など）。
  - ロジック: 出現回数が5回以上、かつ2日以上または2セッション以上使用され、かつ一時的な連投（バーストスコア）ではない場合に **永続キャッシュ** に昇格します。
- **通常の文 (`sentence`)**:
  - 対象: 上記以外の一般的な文。
  - ロジック: 出現回数が4回以上、2日以上使用され、バーストスコアが低い場合に **永続キャッシュ** に昇格します。

### キャッシュのクリーンアップポリシー
ディスク容量の圧迫を防ぐため、自動でバックグラウンドクリーンアップが行われます。

- **容量制限**: キャッシュ全体の容量が **1GB** を超過した場合、使用日時が古い一時キャッシュ（`temp`）から優先的に物理削除されます。
- **有効期限 (TTL)**:
  - **一時キャッシュ**: 最終使用から **7日間** 未使用で自動削除。
  - **永続キャッシュ**: 最終使用から **90日間** 未使用で自動削除。

## 設定ファイル (`config.json`)

アプリケーションの実行ディレクトリに自動生成される `config.json` で詳細な設定を行えます。

主要な設定項目：
- `youtube_api_key`: YouTube Live チャットの取得に必要な YouTube Data API v3 の API キー。（取得先: [Google Cloud Console](https://console.cloud.google.com/apis/library/youtube.googleapis.com)）
- `youtube_url`: 接続対象の YouTube 配信の URL または Video ID（`debug` と入力するとダミーコメントの送信テストモードになります）。
- `tts_engine`: 使用する TTS エンジン名（`voicevox` / `coeiroink` / `bouyomichan` / `supertonic_lightweight` / `supertonic`）。
- `skip_history`: 起動時に、接続前の過去ログコメントの読み上げをスキップするかどうか（`true` 推奨）。
- `read_super_chat`: スーパーチャット等の有料イベントメッセージを読み上げるかどうか。
- `read_blocks`: 読み上げテキストのフォーマットブロック。デフォルトでは `[{"type": "message"}]`（本文のみ）。`{"type": "author"}`（投稿者名）や `{"type": "text", "value": "さん。"}`（固定文字）を組み合わせることで、「〇〇さん。こんにちは」のようなカスタム読み上げが可能です。
- `comment_popout`: コメントウィンドウの PiP（独立表示）状態。

## 辞書データの管理

ユーザー辞書データは `dict/` ディレクトリ配下に JSON 形式で保存されます。
- 初期起動時に `dict/デフォルト.json` が自動生成されます。
- 新しい JSON ファイルを `dict/` に追加することで、独自の辞書グループを追加可能です。
- アプリの設定ダイアログからは、JSON、CSV、および TXT（タブ区切り）形式の既存辞書ファイルをインポートしてマージすることができます。
