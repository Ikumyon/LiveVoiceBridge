# LiveVoiceBridge 使用方法・マニュアル

本ドキュメントでは、LiveVoiceBridge のセットアップ方法から基本的な起動・操作手順、各種機能の詳しい使い方について解説します。

---

## 1. セットアップ（事前準備）

### 依存パッケージのインストール
本アプリケーションは Python 3.14+ で動作します。プロジェクトの仮想環境（`venv`）を準備した上で、必要なライブラリをインストールしてください。

```powershell
# 仮想環境の作成と有効化（例）
python -m venv venv
./venv/Scripts/activate  # Windowsの場合

# 依存ライブラリのインストール
pip install -r requirements.txt
```

### ローカル音声合成（TTS）モデルの配置
Supertonic または Supertonic Lightweight を使用する場合は、あらかじめ以下のディレクトリにモデルアセットが配置されているか確認してください。

- **Supertonic**: `models/supertonic-3/`
- **Supertonic Lightweight**: `models/sherpa-onnx-supertonic-3-tts-int8-2026-05-11/`

*(※Supertonic Lightweight のモデルは、設定画面から直接ダウンロードすることも可能です。ダウンロードする際にはライセンスの確認と同意が必要となります。モデルおよび音声データの使用条件については、ダウンロード前に表示されるダイアログのリンク先（GitHub等）をご確認ください。)*

---

## 2. アプリケーションの起動

Windows の場合は、スタート用のスクリプトまたは直接 `main.py` を呼び出して起動します。

```powershell
# スクリプト経由で実行（推奨）
./run_windows.ps1

# または直接起動
python main.py
```
*※Windows環境では、`python main.py` を実行した場合でも、ワークスペース直下に `venv/Scripts/python.exe` があれば、自動的にその仮想環境の Python プロセスへ切り替えて再起動する仕組みが内蔵されています。*

---

## 3. 初期設定（APIキーとTTSの設定）

起動後、まずYouTube接続と読み上げに必要な初期設定を行います。

1. **設定ダイアログを開く**:
   - メインウィンドウ右上の「⚙️（設定ボタン）」をクリックするか、メニューの「ツール」→「設定」を選択します。
2. **YouTube Data API Key の入力**:
   - 「YouTube設定」の項目にある API Key 欄に、取得した YouTube Data API v3 のキーを入力します。
   - 入力欄の右側にある外部リンクボタンをクリックすると、APIキーの取得先である [Google Cloud Console](https://console.cloud.google.com/apis/library/youtube.googleapis.com) にブラウザでアクセスできます。
   - *(※取得手順：プロジェクトの作成または選択 ➜ 「YouTube Data API v3」を検索して有効化 ➜ 「認証情報」から「API キー」を作成)*
   - *※ローカルデバッグモードのみを使用する場合は、空欄でも構いません。*
3. **TTS（音声合成）エンジンの選択**:
   - 使用したい TTS エンジン（VOICEVOX、COEIROINK、棒読みちゃん、Supertonic、Supertonic Lightweight）を選択します。
   - **VOICEVOX / COEIROINK / 棒読みちゃん**:
     - 既に外部で起動している場合はその URL/ポート を指定します。
     - 自動起動させたい場合は、それぞれの「実行ファイルのパス」を指定しておくと、アプリ起動時にバックグラウンドで自動起動されます。
   - **Supertonic**:
     - 推論デバイス（`cpu` など）や推論ステップ数を設定します。
4. **設定の保存**:
   - 「OK」ボタンを押して設定を保存します（キャンセルした場合は、辞書データ等も含めて設定変更前の状態に自動ロールバックされます）。

## 4. 基本的な操作手順

### YouTube 配信コメントの読み上げ開始
1. メインウィンドウの「YouTube URL または Video ID」入力欄に、接続したいライブ配信の URL または動画IDを入力します。
2. 「開始」ボタンをクリックします。
3. 接続に成功すると、ステータスバーが「接続中。コメント待機中です。」に変わり、配信チャットに投稿されたコメントがリアルタイムに取得され、自動で読み上げられます。

### コメントの読み上げ停止
- 読み上げを終了する場合は、「停止」ボタンをクリックします。接続が切断され、ステータスが「停止中」になります。

### テスト送信モード（デバッグ）
YouTube Liveに接続せず、ローカル環境だけで読み上げや辞書のテストを行いたい場合に使用します。
1. 入力欄に **`debug`** と入力して「開始」ボタンをクリックします。
2. デバッグモードで起動すると、画面に「テスト送信」ボタンが出現します。
3. 「テスト送信」ボタンをクリックし、ダイアログに読ませたいテキストを入力すると、ダミーコメントとして処理され、即座に音声が合成・再生されます。
4. このテスト送信でも、コメントコマンド（`速度(150)` や `再生(se.wav)` など）の動作を確認できます。

### ログとコメントのクリア
- 画面に表示されているコメント一覧および動作ログテキストをリセットしたい場合は、「ログクリア」ボタンをクリックします。

## 5. 各種機能の詳しい使い方

### 📎 ピクチャーインピクチャー（PiP）コメントウィンドウ
コメント表示部分をメインウィンドウから切り離し、デスクトップ上の好きな位置に最前面で常時表示（半透明表示も可能）させることができます。

- **切り替え方法**:
  - メインウィンドウ上部にある PiP ボタン（テレビや画面のアイコン）をクリックすると、コメントリストが独立した `CommentWindow` に移動します。もう一度クリックするとメインウィンドウのタブに戻ります。
- **ウィンドウの調整**:
  - 分離されたコメントウィンドウは、端をドラッグすることで自由にサイズを変更できます。また、位置も自動保存されます。
  - 設定ダイアログの「コメントウィンドウ不透明度」スライダーを調整することで、ウィンドウの背景を半透明にし、配信画面等に重ねやすくなります。

### 📖 ユーザー辞書と単語インポート
特定の単語の誤読を防ぐための辞書機能です。

- **インポート手順**:
  1. 設定ダイアログの「辞書」タブを開きます。
  2. 「インポート」ボタンをクリックし、手持ちの辞書ファイルを選択します。
     - **対応フォーマット**:
       - **JSON**: `[{"word": "単語", "reading": "よみがな"}]` または `{"単語": "よみがな"}` 形式
       - **CSV**: カンマ区切りのテキスト（1列目: よみがな、2列目: 単語、3列目: 品詞、4列目: メモ）
       - **TXT**: タブ区切りのテキスト（辞書ツール形式）
  3. インポートすると、既存の辞書グループにマージされ、読み上げ時に自動的に適用されます。
- **配信中の教育**:
  - 視聴者がチャット欄で `教育(単語=よみがな)` とコメントすることで、自動的にユーザー辞書の「配信コメント」グループに登録されます。

### 🎵 音声エフェクトと効果音（SE）の利用
- **効果音（SE）の準備**:
  - ワークスペースの `sounds/` ディレクトリ（存在しない場合は初回起動時に自動生成されます）に、再生したい音源ファイル（`.wav` など）を配置します。
  - 視聴者がコメント内で `再生(ファイル名)`（例: `再生(applause.wav)`）と打つことで、読み上げと同時に効果音が再生されます。
- **エコー・山びこ**:
  - `エコー(50)` または `やまびこ(50)` コマンドをチャットに含めることで、自製の波形ディレイ効果が付与されます。
  - やまびこは3回リピート（減衰あり）、エコーは1回のリピート効果になります。
- **定位（パンニング）**:
  - `左)` や `右)` コマンドで、音声の出力チャンネルを左側のみ、右側のみに瞬時に切り替えることができます。`両)` で左右均等に戻ります。
