# LiveVoiceBridge 開発エージェント指示書 (agent.md)

本ドキュメントは、本プロジェクト（LiveVoiceBridge）において、AIコーディングエージェントが開発・リファクタリング・機能追加を行う際の基本方針、設計思想、および厳守すべき制約ルールを定義したものです。
以降の開発においては、必ず本ドキュメントの方針に準拠してください。

---

## 1. プロジェクト概要
* **名称**: LiveVoiceBridge
* **目的**: YouTube Live のチャット（コメント）をリアルタイムに取得し、音声合成エンジン（TTS）を利用して自動で読み上げるデスクトップアプリケーション。
* **主要技術スタック**:
  - 言語: Python 3.x
  - UIフレームワーク: PySide6 (Qt for Python)
  - 音声合成連携: VOICEVOX / COEIROINK 等の HTTP API 経由による連携

---

## 2. 基本設計方針（アーキテクチャ）

### ① 多態性（ポリモーフィズム）の徹底
* 特定の音声合成エンジン（VOICEVOX、COEIROINK 等）に依存する具象ロジックを、メインの処理（`main.py`）や設定画面（`settings_dialog.py`）に直接記述してはならない。
* **抽象化レイヤー**:
  - すべての音声合成エンジンは [core/tts_engines.py](file:///c:/localfile/開発/LiveVoiceBridge/core/tts_engines.py) の `BaseTTSEngine` を継承して具象クラスを作成する。
  - メインロジックからは、具象クラスではなく `BaseTTSEngine` のインターフェースを通じて操作する。
* **ファクトリパターンの利用**:
  - クラスの取得、インスタンスの生成、設定マイグレーションの呼び出しなどは、すべて [core/tts_factory.py](file:///c:/localfile/開発/LiveVoiceBridge/core/tts_factory.py) を経由して行う。

### ② 責務の分離 (Separation of Concerns)
各ファイルの役割を明確にし、肥大化を防ぐ。
* **アプリ起動・メインUI制御**: [main.py](file:///c:/localfile/開発/LiveVoiceBridge/main.py)
  - ウィンドウの生成、メイン画面のイベントバインド、スレッドのライフサイクル管理に専念する。
* **辞書I/O処理**: [core/dictionary.py](file:///c:/localfile/開発/LiveVoiceBridge/core/dictionary.py)
  - 読み替え辞書ファイル（JSON）の読み込み、保存、追加、削除の純粋なファイル操作ロジックのみを置く。
* **バックグラウンド処理**: [core/workers.py](file:///c:/localfile/開発/LiveVoiceBridge/core/workers.py)
  - コメント受信スレッド（`ChatStreamWorker`）や音声読み上げスレッド（`SpeechWorker`）のロジックを配置する。
* **UIグラフィック補助**:
  - 画像の切り抜きなどの純粋なグラフィック関連のユーティリティ処理は、画面クラスの内部メソッドにせず、クラス外のスタンドアロンな関数として定義して見通しを良くする。

---

## 3. コーディング＆ネーミング規約

* **汎用的な命名の利用**:
  - 変数名、ウィジェット名、設定キー、関数名などに特定のエンジン名（例: `voicevox_url`, `testVoicevoxButton`）を埋め込まず、`tts_url`, `ttsTestButton` などの汎用的な接頭辞（`tts`）を使用する。
* **コメント・ドキュメント**:
  - ソースコード中の既存のコメント、docstringは、変更対象と直接関係ない限り、破壊せずに維持・保護すること。

---

## 4. 設定ファイルの管理ルールと互換性の排除

* **新しい設計方式への完全な一本化**:
  - 開発中のプロジェクトであるため、過去の古いコードや古い設定項目との後方互換性は一切考慮しない。
  - 設計や機能追加は「新しい方式のみ」で行い、古い仕様に合わせるためのフォールバックコードや妥協のロジックは一切含めない。
* **新設定構造の維持**:
  - `config.json` はエンジンごとにネストされた新構造（`"voicevox": { "url": "...", ... }`）で管理する。
* **既存設定の移行と古いキーの廃止**:
  - 既存のフラットな古い設定項目（`voicevox_url` など）からの移行処理は起動時に行い、移行後は `pop` などで古いキーを**完全に削除**する。
  - 保存時には新しいネスト構造のみを書き出し、古いキーの二重書き出しなどは絶対に行わない。

## 5. エージェントの行動制約・ルール

### ① 日本語の徹底
* ユーザーへの回答、およびAIエージェントが作成・更新する各種ドキュメント（`implementation_plan.md`、`task.md`、`walkthrough.md`、`agent.md` など）は、**すべて必ず日本語**で出力すること。

### ② 勝手なコード変更の禁止
* ユーザーからの明確な指示がない限り、ソースコードを書き換えないこと。

### ③ 新しいライブラリ導入時の確認手順
プロジェクトに新しいライブラリを追加する際は、インストールコマンドを実行する前に、必ず現在の実行環境に既に該当パッケージがインストールされているか（例: `pip show <パッケージ名>` など）を事前に確認すること。
* 既にインストールされていることが確認された場合は、無駄なインストールプロセスを実行せず、次の開発ステップへ進むこと。
